# 有人做了一个「OpenClaw 暴露面监控地图」，我看到了 50+ 台服务器在裸奔

来源：Hacker News

昨天，Hacker News 上出现了一个有趣的项目：**OpenClaw Exposure Watchboard**（OpenClaw 暴露面监控看板）。

简单来说，就是有人做了一个全球地图，把那些「暴露在公网上、没有任何保护」的 OpenClaw 服务器都标记了出来。

## 我看到了什么？

根据这个监控平台的数据，目前全球有 **50+ 台 OpenClaw 服务器** 暴露在公网，包括：

| 国家 | 服务器数量 |
|------|-----------|
| 🇺🇸 美国 | 最多 |
| 🇨🇳 中国 | 腾讯云、阿里云、百度云 |
| 🇸🇬 新加坡 | 腾讯云 |
| 🇩🇪 德国 | Hetzner |
| 🇫🇮 芬兰 | Hetzner |
| 🇫🇷 法国 | Hostinger、Ionos |

**云服务商分布**：DigitalOcean、Hetzner、腾讯云、阿里云、百度云、Oracle、AWS...

## 问题来了：这些服务器在「裸奔」

监控显示，很多服务器存在以下问题：

### 1. 没有开启认证
有些服务器的 `auth_required` 是 **No**，意味着任何知道地址的人都能直接访问。

### 2. 凭证泄露
一些服务器的 API 密钥、登录凭证已经被泄露到公开环境，存在**严重安全风险**。

### 3. 被黑客组织盯上
部分 IP 被标记了 **APT（高级持续性威胁）组织关联** —— 也就是说，某些国家级黑客已经在关注这些暴露的服务器。

> 这不是危言耸听。数据显示，这些 IP 中关联了 **CVE 漏洞**、**恶意软件家族**，甚至有 **Lazarus Group**（朝鲜黑客组织）相关标记。

## 这说明什么？

### 1. OpenClaw 真的在「出圈」

50+ 台公开服务器，说明越来越多的人在使用 OpenClaw。这是好事，说明工具有价值。

### 2. 安全意识严重不足

很多人部署 OpenClaw 后，直接把端口 18789 暴露到公网，却没有：
- 开启认证
- 配置防火墙
- 使用 VPN/Tailscale 访问

这就是把家门敞开给小偷看。

## 如何保护自己？

### 方案一：内网访问（推荐）

不要把 OpenClaw 暴露到公网。回家后通过 **Tailscale** 或 **VPN** 访问：

```bash
# 安装 Tailscale
curl -fsSL https://tailscale.com/install.sh | sh
```

### 方案二：必须暴露公网？

那就做好以下防护：

1. **开启认证** — 在配置中设置 `authRequired: true`
2. **使用强密码** — 定期更换 API Key
3. **防火墙规则** — 只允许可信 IP 访问
4. **定期检查** — 查看是否有异常登录

### 方案三：换端口

别用默认的 18789 端口，改成一个随机的高端口，降低被扫描到的概率。

## 最后号

这个监控平台的出现，其实给整个社区提了个醒：

> **工具越强大，普及越广，安全问题就越重要。**

如果你正在用 OpenClaw，赶紧检查一下自己的部署有没有暴露风险。
